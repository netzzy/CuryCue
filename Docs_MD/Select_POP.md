# Select POP

##   
  
Summary

The Select POP takes individual attributes from its single input and outputs the attributes unchanged. It is used frequently so you can work on attributes separately, and then after processing the attributes, you can combine them back together downstream with an [Attribute Combine POP](<./Attribute_Combine_POP.md> "Attribute Combine POP"). 

The number of points does not change. The data in the selected attributes does not change. The topology does not change either - it outputs the same primitives and vertices as the input. 

Because attributes of operators in the POPs family that are not changed in a POP don't actually copy their data or allocate any more memory (they are simply a reference to the POP that does have the data), selecting attributes and combining them back does not make anything faster or memory-efficient. It's mostly a workflow preference that is more similar to the CHOPs workflow. Middle-mouse click on the Select POP node to see the info pop-up, and note that the GPU memory consumed is 0. 

Instead of a wired input you can use the POP parameter to reference another POP in the current or any other network. 

Note: Unlike the [Attribute Combine POP](<./Attribute_Combine_POP.md> "Attribute Combine POP") that increases the number of point attributes and does not change the number of points, the [Merge POP](<./Merge_POP.md> "Merge POP") increases number of points by merging primitives and their vertices together with the point lists of each input. For example, it merges the`P`attribute of all the points together to make a longer point list. 

See also [Attribute Combine POP](<./Attribute_Combine_POP.md> "Attribute Combine POP"), [Math Mix POP](<./Math_Mix_POP.md> "Math Mix POP") and [math Combine POP](<./Math_Combine_POP.md> "Math Combine POP") (can also extract individual attributes), [Merge POP](<./Merge_POP.md> "Merge POP"), [Topology POP](<./Topology_POP.md> "Topology POP")
